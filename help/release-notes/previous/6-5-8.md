# [!DNL Adobe Experience Manager] 6.5.8.0 {#experience-manager-6580}

[!DNL Adobe Experience Manager] 6.5.8.0包括自2019年4月发布6.5版以来发布的新功能、关键客户请求的增强以及性能、稳定性和安全性改进。 服务包安装在 [!DNL Adobe Experience Manager] 6.5.

中引入的主要特性和增强功能 [!DNL Adobe Experience Manager] 6.5.8.0包括：

<!-- TBD:
* Using the Connected Assets functionality, it is now possible to connect up to 3 [!DNL Sites] instances with 1 [!DNL Assets] instances. The configuration user interface now allows the administrators to provide the details of these [!DNL Sites] instances. -->

* 使用时 [“连接的资产”功能](/help/assets/use-assets-across-connected-assets-instances.md)，您现在可以查看所有 [!DNL Sites] 使用资产的页面。 这些对资产的引用可在资产的 [!UICONTROL 属性] 页面。 这使管理员、营销人员和库管理员可以全面了解资产使用情况，从而更好地跟踪、管理和品牌一致性。

* 删除网页中引用的资产时， [!DNL Experience Manager] [显示警告](/help/assets/use-assets-across-connected-assets-instances.md#asset-usage-references). 您可以强制删除引用的资产，或者检查和修改在中显示的引用。 [!DNL Properties] 页面时，不会显示重复项。 单击引用将打开本地和远程 [!DNL Sites] 页数。

* 使用对可用于转出的Live Copy页面进行排序 [!UICONTROL 名称]， [!UICONTROL 上次修改日期，] 和 [!UICONTROL 上次转出日期] 属性。

* 内置存储库(Apache Jackrabbit Oak)已更新至1.22.6。 <!-- TBD: Mention the version -->

以下是中提供的修复列表 [!DNL Experience Manager] 6.5.8.0版本。

## [!DNL Sites] {#sites-6580}

* 将页面移动到Blueprint时，链接的目标不会更新(NPR-35724)。
* 基于Tizen的播放器在某些浏览器上无法进行身份验证。 对于不支持samesite=none属性的浏览器，会出现此问题(NPR-35589)。
* 解锁的响应容器不显示允许的组件(NPR-35565)。
* 创建新添加页面的Live Copy时，语言母版会为每个域创建两个副本(NPR-35545)。
* SCR组件注册表中的死锁，当许多线程由于以下原因被阻止时 `org.apache.felix.scr.impl.ComponentRegistry` 计时器。 因此， [!DNL Experience Manager] 停止无限期响应(GRANITE-33125、FELIX-6252)。
* 在侧边栏中搜索特定资产时，结果会包含一些未搜索的资产(NPR-35524)。
* 为Experience Manager实例启用SSL时，上下文路径会被删除(NPR-35477)。
* 创建列表时，添加一些文本作为第一个元素，添加一个表作为第二个元素，并在表中添加一个列表，则父列表会发生扭曲(NPR-35465)。
* 对连续列表项目使用不同的插件时，额外使用 <br> 标记将被添加到列表项目(NPR-35464)。
* 当列表置于两段之间时，您无法将表添加到列表中(NPR-35356)。
* 开始将AEM实例从AEM 6.3升级到AEM 6.5时，升级实例的启动时间会更长(NPR-35323)。
* 复制包含中括号()的AEM资源时。 在名称中，复制失败(GRANITE-27004、NPR-35315)。
* 将标题添加到富文本编辑器时，段落按钮被禁用(NPR-35256)。
* 将项目添加到现有列表时，会删除后续的可折叠或切换列表(NPR-35206)。
* 选择“转出”页面选项后，将显示一个包含所有可用活动副本的对话框，并发生自动转出。 无需用户操作，页面活动副本即可转出到所有地理位置(NPR-35138)。
* 使用包括子项选项时，管理发布选项不会列出所有页面。 仅列出22页(NPR-35086)。
* 编辑策略时，文本组件不保留策略更改(NPR-35070)。
* 在编号列表中缩进某些项目时，所有项目都会保持相同的编号，但具有相同缩进的项目编号应该从1开始(CQ-4313011)。
* 启用缩小功能后，您将无法编辑任何页面或组件。 安装AEM 6.5 Service Pack 7 (CQ-4311133)后开始出现问题。
* 全搜索过滤器和资产过滤器返回不相关或没有结果(CQ-4312322、NPR-35793)。
* 当多个页面同时访问客户端库时，HTML库管理器无法加载客户端库。 这会导致页面呈现不正确(NPR-35538)。
* 当您在中设置SSL时，上下文路径将自动删除 [!DNL Experience Manager] (NPR-35294)。
* 单击注销选项后，包管理器不会注销用户(NPR-35160)。

## [!DNL Assets] {#assets-6580}

[!DNL Adobe Experience Manager] 6.5.8.0 [!DNL Assets] 修复了以下问题并提供了以下增强功能。

* 恢复资产的先前版本时，不会在OSGi控制台中触发事件DamEvent.Type RESTORED (NPR-35789)。
* `IndexWriter.merge` 原因 `OutOfMemoryError` 错误，因为智能标记功能会创建 `/oak:index/lucene` 和 `/oak:index/ntBaseLucene` 索引(NPR-35651)。
* 尝试保存时出现错误消息 [!UICONTROL 资产贡献] 键入名称中包含多字节字符的文件夹(NPR-35605)。
* 使用层叠元数据子类型字段时，出现错误的“请填写此字段”错误(NPR-35643)。
* 将现有资产拖动到 [!DNL Assets] 用户界面和创建新版本后，元数据的更改不具有永久性(NPR-34940)。
* 在层叠菜单的元数据架构编辑器中创建规则时， [!UICONTROL 取决于] 选项重复相同的名称(NPR-35596)。
* 编辑后相似性搜索不起作用 [!UICONTROL 资产管理搜索边栏] (NPR-35588)。
* 在文件夹中，如果通过单击在左边栏中打开资产搜索 [!UICONTROL 筛选]，中的筛选条件 [!UICONTROL 状态] > [!UICONTROL 结帐] > [!UICONTROL 已签出] 不起作用(NPR-35530)。
* 如果尝试删除资产的所有智能标记并保存更改，则不会删除标记。 但是，用户界面会显示更改已保存(NPR-35519)。
* 用户无法对列表视图中可排序文件夹内的资产重新排列或排序(NPR-35516)。
* 如果您编辑默认的元数据架构，则资源的 [!UICONTROL 属性] 页面更改为文本字段。 此更改允许不了解此更改的用户添加即时标记，并且标记将作为字符串存储在存储库中(NPR-35478)。
* 在下载资源时，如果您提供的名称没有有效的电子邮件地址，则下载选项不可用。 但是，如果选择了下载对话框中的其他选项，则会启用该按钮，但不会发送电子邮件(NPR-35365)。
* 用户在中编辑资产后无法签入资产 [!DNL Adobe InDesign] 并收到有关缺少权限的错误(NPR-35341)。
* Handlebars JavaScript库已升级到v4.7.6 (NPR-35333)。
* 当您从批量元数据编辑开始时，元数据编辑器界面会按预期停止工作，并取消选择项目，直到单个项目保持选中状态(NPR-35144)。
* 从内单击时，全局导航不会打开正确的控制台 `assets.html` 页面(CQ-4312311)。
* [!DNL Assets] 不显示具有RGB演绎版的RGB的资源的CQ-4310190。
* 此 [!UICONTROL 相关] 菜单中的选项未正确显示在 [!UICONTROL 属性] 页面(CQ-4310188)。
* 如果文档的文件类型过滤器用于搜索资源和创建智能收藏集，则在访问收藏集时不会应用该过滤器。 相反，所有类型的资产都显示在搜索中(NPR-35759)。
* 您不能在Lightbox中将资产从 [!DNL Assets] 用户界面(NPR-35901)。
* 解决命名冲突后创建现有资源的新版本时，原始资源的元数据会被覆盖(CQ-4313594)。
* 使用搜索过滤器或谓词筛选资源搜索，打开资源进行查看或编辑，然后返回搜索结果页面时，过滤器不起作用。 所有搜索到的资源都未经过筛选(NPR-35913)。

## UberJar {#uber-jar}

适用于Experience Manager6.5.8.0的UberJar可在 [Maven中央存储库](https://repo.maven.apache.org/maven2/com/adobe/aem/uber-jar/6.5.8/).

要在Maven项目中使用UberJar，请参阅 [如何使用UberJar](/help/sites-developing/ht-projects-maven.md) 并在项目POM中包含以下依赖项：

```shell
<dependency>
     <groupId>com.adobe.aem</groupId>
     <artifactId>uber-jar</artifactId>
     <version>6.5.8</version>
     <scope>provided</scope>
</dependency>
```

>[!NOTE]
>
>UberJar和其他相关工件可在Maven中央存储库而不是Adobe公共Maven存储库(`repo.maven.apache.org`)。 主UberJar文件将重命名为 `uber-jar-<version>.jar`. 所以，没有 `classifier`，替换为 `apis` 作为值，用于 `dependency` 标记之前。
