# 下一个版本……{#the-next-release}

实施一个项目很少是过程的结束。 后续版本（几乎）始终可以预见，甚至可以在当前实施的最后阶段开始。

无论如何，通过以下方式为下一个版本进行规划会很有用：

* 仔细记录和存储您的测试案例，以便引用或希望重复使用。
* 开发可用作下一个版本的回归测试的测试。
* 整理您或客户通过测试过程获得的全部信息。

  这可能有助于形成下一个版本的要求；例如，因为在测试期间客户对产品的体验有所增加。

由于下一个版本很可能是迁移，因此任何与此相关的信息都将被证明非常有用。
