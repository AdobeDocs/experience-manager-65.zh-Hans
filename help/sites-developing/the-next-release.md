# 下一个版本……{#the-next-release}

项目的实施很少会结束。 随后的版本始终（几乎）可以预见，甚至可以在当前实施的最后阶段开始。

无论如何，通过以下方式规划下一个版本可能会非常有用：

* 仔细记录和存储您的测试用例，以便引用或希望重复使用。
* 开发可用作下一版本回归测试的测试。
* 整理您或客户通过测试过程获得的任何信息。

   这可能有助于形成下一版本的要求；例如，由于客户在测试期间对产品的体验有所增加。

由于下一个版本很可能是迁移，因此任何与此相关的信息都将被证明是有用的。
