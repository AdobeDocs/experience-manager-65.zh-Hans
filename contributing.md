# 向 Adobe Experience Manager 文档贡献内容

感谢您选择为 Adobe Experience Manager 文档贡献内容！Adobe 欢迎来自整个读者社区和 Adobe 员工的内容贡献。

本文介绍了如何贡献内容，并列出了贡献内容时应遵循的准则。

## Adobe 开源项目行为准则

本项目采用 [Adobe 开源项目行为准则](code-of-conduct.md)。如果选择为 AEM 文档贡献内容，即表明您同意遵守本行为准则。请联系 [Grp-opensourceoffice@adobe.com](mailto:Grp-opensourceoffice@adobe.com) 举报不可接受的行为。

## 内容贡献准则

为完善 AEM 文档，我们欢迎任何人贡献内容，但是我们要求贡献的内容尽可能遵守特定的准则和标准。请参阅 [AEM 文档内容贡献准则](guidelines.md)，了解更多信息。

## 如何贡献内容

我们希望让您能够尽可能轻松地贡献内容。您可以通过两种方式贡献内容：

* [创建问题](#create-an-issue)
* [创建拉取请求](#create-a-pull-request)

至于选择哪种方式，取决于您更喜欢采用何种方式在 GitHub 中编辑文档。

>[!NOTE]
>
>您提交的内容将受 [Adobe 使用条款](https://www.adobe.com/cn/legal/terms.html)的约束。

### 创建问题 {#create-an-issue}

对于当前的文档或者内容陈述方式，您是否有任何建议、发现或疑问？然后创建一个问题。如果您有要贡献的内容，但不方便直接编辑文档或提交拉取请求，也可以通过提交问题进行参与。

通过提交问题可让您能够轻松描述对文档的改进想法，AEM 文档团队成员可以对其进行审核。只需在任意页面右侧导航面板顶部点击&#x200B;**记录问题**&#x200B;链接即可创建问题。

### 创建拉取请求 {#create-a-pull-request}

如果您发现了问题并且确切知道如何解决该问题，请考虑创建拉取请求。通过拉取请求，您可以直接编辑文档，随后会由 AEM 文档团队成员审核并决定是否采纳。您只需点击任意页面右侧导航面板顶部的&#x200B;**编辑此页面**&#x200B;链接，即可提交建议的修改并创建拉取请求。

所有拉取请求都必须附有已签署的内容[贡献者许可协议（简称 CLA）](https://opensource.adobe.com/cla.html)。这将允许 Adobe 在项目中再次分发您的贡献。您只需提交一次 Adobe CLA 即可，因此，如果您以前已经提交过，则无需再次提交！

### 来自 Adobe 员工的内容贡献

如果您是 Adobe 员工并且有内容要贡献，使用公司 git 中的 `AdobeDocs` 组织。

我们欢迎所有贡献，并会审核公共存储库中的贡献；不过，Adobe 员工的拉取请求在企业存储库中更易处理且可更快获得批准。请确保任何专有信息只在公司存储库中讨论。
